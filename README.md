## About

I design end-to-end ML systems.

Data → Features → Training → Evaluation → API/Agent → Deployment → Observability.

I care about:
- Training–serving parity  
- Retrieval system design  
- Controlled LLM tool-calling  
- Failure-mode analysis  
- Reproducible pipelines  

I prefer systems that are inspectable, versioned, and deployment-aware.

---

## Flagship Systems

**Agentic Research Assistant**  
LangGraph-based reasoning system with tool-calling, persistent memory, RAG-as-a-tool, and LangSmith observability.  
FastAPI + Streamlit + SQLite.

**Regression ML — End-to-End**  
Time-aware validation, unified train/serve pipeline, Optuna tuning, MLflow tracking, Dockerized services, AWS ECS deployment, CI/CD automation.

**GPT from Scratch (PyTorch)**  
Decoder-only transformer trained on Shakespeare (~1.1M chars).  
Manual implementation of tokenization, attention, embeddings, and sampling.  
Perplexity: 3.02 | Accuracy: 64.9%.

**Reasona — Self-Improving RAG**  
HyDE → Critic → Self-edit → Vector update.  
Exploring incremental retrieval improvement.
